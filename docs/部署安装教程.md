# FIX-ING 部署安装教程

> 通用设备维保平台 · v0.3
> 本教程面向**从零开始**在一台服务器（或本机）上把系统跑起来的场景。跟着做即可，不需要懂 Java。

---

## 一、这套系统需要什么

FIX-ING 是一个打包好的单文件程序（一个 `.jar`），运行它需要三样东西：

| 依赖 | 作用 | 版本要求 |
|------|------|----------|
| **JDK（Java 运行环境）** | 运行程序本体 | **17 或更高**（推荐 17 / 21） |
| **MySQL** | 存所有业务数据（工单、客户、设备…） | 5.7+ 或 8.x |
| **Redis** | 存登录令牌黑名单、看板缓存 | 5.0+ |

> 前端（Vue3 页面）已经打包进了 jar 里，**不需要单独部署前端、不需要装 Node.js**。

---

## 二、准备环境（三选一，按你的系统）

### macOS（用 Homebrew）
```bash
brew install openjdk@17 mysql redis
brew services start mysql
brew services start redis
```

### Ubuntu / Debian
```bash
sudo apt update
sudo apt install -y openjdk-17-jre-headless mysql-server redis-server
sudo systemctl start mysql redis-server
sudo systemctl enable mysql redis-server   # 开机自启
```

### CentOS / RHEL
```bash
sudo yum install -y java-17-openjdk redis
# MySQL 按官方源安装；或用 MariaDB：sudo yum install -y mariadb-server
sudo systemctl start redis mysqld
```

**验证环境就绪：**
```bash
java -version      # 显示 17 或更高即可
mysql --version    # 能显示版本
redis-cli ping     # 返回 PONG
```

## 三、初始化数据库

> 发布包里的 `sql/` 目录有两个脚本：`schema.sql`（建库建表）、`seed.sql`（演示数据/初始账号）。

**1. 创建数据库并导入表结构**（会自动创建名为 `fixing` 的库）：
```bash
mysql -u root -p < sql/schema.sql
```

**2. 导入初始数据**（包含 5 个演示账号、权限配置、示例客户/设备）：
```bash
mysql -u root -p < sql/seed.sql
```

> ⚠️ `seed.sql` 会清空并重建演示数据，**只在首次部署时执行**。正式上线后不要重复跑，否则会清掉真实数据。
> 如果只想要一个干净的空系统，见文末「附录 B：只建表不灌演示数据」。

## 四、启动程序

发布包里有个 `start.sh`（Windows 用 `start.bat`）。**先按你的实际情况改里面的数据库密码**，然后运行：

```bash
bash start.sh
```

看到 `Started FixingApplication in x.x seconds` 就是启动成功了。

**打开浏览器访问：** http://服务器IP:8080

> 本机部署就是 http://localhost:8080

### 配置项（通过环境变量覆盖，不用改 jar）

`start.sh` 本质上是设置几个环境变量后运行 jar。可配置的全部项：

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `DB_URL` | `jdbc:mysql://localhost:3306/fixing?...` | 数据库连接地址（换主机/端口/库名时改它） |
| `DB_USERNAME` | `root` | 数据库用户名 |
| `DB_PASSWORD` | `1234` | **数据库密码（务必改成你自己的）** |
| `REDIS_HOST` | `localhost` | Redis 地址 |
| `REDIS_PORT` | `6379` | Redis 端口 |
| `REDIS_PASSWORD` | （空） | Redis 密码，没设密码就留空 |
| `JWT_SECRET` | 内置开发值 | **登录令牌签名密钥，上线必须换成随机长串**（≥32 字符） |
| `SERVER_PORT` | `8080` | 程序监听端口 |

手动启动示例（不用 start.sh 时）：
```bash
export DB_PASSWORD='你的数据库密码'
export JWT_SECRET='换成一串你自己的随机字符至少32位xxxxxxxxxx'
java -jar fixing-admin-0.3.0.jar
```

## 六、让它在后台常驻（生产环境）

前面的 `bash start.sh` 关掉终端就停了。让它常驻有两种方式：

### 方式 A：nohup（最简单）
```bash
nohup bash start.sh > fixing.log 2>&1 &
# 查看日志：tail -f fixing.log
# 停止：找到进程号 kill 掉  ->  lsof -ti:8080 | xargs kill
```

### 方式 B：systemd（Linux 推荐，开机自启+崩溃自拉起）
创建 `/etc/systemd/system/fixing.service`：
```ini
[Unit]
Description=FIX-ING 维保平台
After=mysql.service redis-server.service

[Service]
WorkingDirectory=/opt/fixing
Environment=DB_PASSWORD=你的数据库密码
Environment=JWT_SECRET=你的随机密钥至少32位
ExecStart=/usr/bin/java -jar /opt/fixing/fixing-admin-0.3.0.jar
Restart=always
User=fixing

[Install]
WantedBy=multi-user.target
```
然后：
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now fixing
sudo systemctl status fixing     # 查看状态
journalctl -u fixing -f          # 查看日志
```

## 七、常见问题排查

| 现象 | 原因 & 解决 |
|------|-------------|
| 启动报 `Access denied for user` | 数据库密码不对 → 改 `DB_PASSWORD` |
| 启动报 `Unknown database 'fixing'` | 没导入 schema.sql → 回到第三步 |
| 启动报 `Unable to connect to Redis` | Redis 没启动 → `redis-cli ping` 应返回 PONG |
| 页面能开但登录转圈/报错 | 后端没起来或端口不对 → 看日志、确认 8080 |
| 端口 8080 被占用 | 换端口：`export SERVER_PORT=9090` 再启动 |
| `java: command not found` | JDK 没装或没进 PATH → 回到第二步 |
| 上传的图片/视频看不了 | 程序目录下会自动建 `uploads/` 存文件，确保有写权限 |

## 附录 A：升级到新版本

1. 停掉旧程序
2. 换上新的 jar
3. 如果新版本带了 `sql/upgrade-x.x.sql` 升级脚本，按说明执行（**不要**重新跑 seed.sql）
4. 重新启动

## 附录 B：只建表不灌演示数据（干净上线）

只执行 `schema.sql` 会得到一个没有任何账号的空库 —— 但那样连不进去。建议：
1. 先正常导入 `seed.sql`
2. 用 `super` 登录，建好自己的真实账号
3. 到「平台·人事」停用所有演示账号

这样既有可登录的超管，又不残留演示数据。
